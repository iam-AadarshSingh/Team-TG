what is prisma scheme and env local

Good question — let me break both down since you'll run into them right away.

Prisma Schema
Prisma is a tool that sits between your Next.js code and your database (Postgres, in your case). Instead of writing raw SQL, you describe your data in a file called schema.prisma, and Prisma generates the code to read/write that data safely with full TypeScript autocomplete.

The schema is basically a blueprint of your database tables. For your project, it would define things like:

prisma
model Creator {
  id        String   @id @default(cuid())
  inGameName String
  realName  String
  game      String
  bio       String?
  youtube   String?
  instagram String?
  twitch    String?
  discord   String?
  order     Int      @default(0)
  active    Boolean  @default(true)
  createdAt DateTime @default(now())
  updatedAt DateTime @updatedAt
}
This one block automatically gives you a Creator table in your database, plus TypeScript functions like prisma.creator.create(...), prisma.creator.findMany(...), prisma.creator.update(...) — which is exactly what your admin panel's "add/edit/delete creator" buttons would call.

You'd have one model block like this for each thing you need to manage: Creator, Sponsor, Game, CareerOpening, ContactSubmission, etc.

.env.local
This is just a plain text file in your project root (.env.local) where you store secret values that your code needs but that should never be committed to GitHub — API keys, database connection strings, auth secrets.

Example of what yours would eventually look like:

DATABASE_URL="postgresql://user:password@host:5432/dbname"
NEXTAUTH_SECRET="some-random-long-string"
NEXTAUTH_URL="http://localhost:3000"
NEXT_PUBLIC_SUPABASE_URL="https://xxxx.supabase.co"
NEXT_PUBLIC_SUPABASE_ANON_KEY="xxxxxxxx"
EMAILJS_SERVICE_ID="xxxxxxxx"
EMAILJS_TEMPLATE_ID="xxxxxxxx"
EMAILJS_PUBLIC_KEY="xxxxxxxx"
Key things to know:

.gitignore (which create-next-app already sets up) excludes .env.local from git automatically — so it stays private on your machine.
On Vercel, you re-enter these same values manually in Project Settings → Environment Variables in the dashboard, since Vercel obviously can't read a file that never got pushed to GitHub.
NEXT_PUBLIC_ prefix matters: anything with that prefix gets exposed to the browser (fine for things like a public Supabase URL). Anything without it stays server-only (correct for secrets like your database password).
